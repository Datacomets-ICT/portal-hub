## 1. ระบบนี้คืออะไร?

**ระบบ IT Ticket** คือเว็บที่ให้พนักงานในบริษัทใช้**แจ้งปัญหา IT** (เช่น คอมพัง, ปริ้นไม่ออก, ขอติดตั้งโปรแกรม) แทนการโทรหา IT หรือเดินไปหาเอง — **ใช้ช่องทางเดียวจบ ติดตามสถานะได้ ไม่หลุดกระจาย**

### 1.1 ทำไมต้องมี

- เดิมพนักงานแจ้งปัญหาผ่านหลายช่องทาง (โทร / ไลน์ / บอกปากเปล่า) — ข้อมูลหาย, IT ไม่รู้ว่าใครรออะไรอยู่บ้าง
- ไม่มีการวัดว่า**งานเสร็จตรงเวลา**หรือไม่
- พนักงานไม่รู้ว่าเรื่องที่แจ้งอยู่ในขั้นไหน
- ทีม IT ไม่มีฐานข้อมูลประวัติปัญหาให้ย้อนดู

### 1.2 เป้าหมาย

1. **รวมช่องทางแจ้งปัญหา IT ให้เป็นที่เดียว** — ไม่ต้องโทร ไม่ต้องไลน์ แจ้งผ่านเว็บได้เลย
2. **ให้ AI ช่วยพนักงานเปิด Ticket** — พนักงานไม่ต้องกรอกฟอร์มเอง ให้คุยกับบอท บอทจะถามข้อมูลให้เอง
3. **แยกสิทธิ์ดูข้อมูลตามหน้าที่** — ลูกน้องเห็นเฉพาะงานตัวเอง / หัวหน้าเห็นงานในทีม / ผู้บริหารเห็นทั้งบริษัท
4. **วัดผลการทำงานของทีม IT ได้** — เช่น งานด่วนต้องเสร็จใน 2 ชม. ระบบจะเตือนเมื่อใกล้ครบเวลา
5. **เก็บความรู้ไว้ใช้ต่อ** — เวลา IT แก้ปัญหาเสร็จ จะบันทึกวิธีแก้ไว้ AI จะนำไปตอบคำถามในอนาคต

### 1.3 ชี้วัดความสำเร็จ

- พนักงานเปิด Ticket ผ่าน AI Chatbot ≥ 80% (แทนการกรอกฟอร์มเอง)
- Ticket เสร็จภายในเวลาที่กำหนด ≥ 85%
- งานด่วน (Urgent) ต้องมีการตอบกลับภายใน 2 ชม.

## 2. มีใครใช้ระบบนี้บ้าง?

### 2.1 โครงสร้าง 4 ระดับ

```
🟣 ผู้จัดการใหญ่ (คุณเบิร์ด)
   │ เห็นได้ทุกเรื่อง ทุกบริษัท
   │
   ├── 🟠 หัวหน้าทีม IT ฝั่ง ICT (คุณวี)
   │       เห็นเฉพาะเรื่องของ ICT
   │       ลงมือแก้เอง (ไม่มีลูกน้อง)
   │
   └── 🟠 หัวหน้าทีม IT ฝั่ง Comets (คุณแชมป์)
           เห็นเฉพาะเรื่องของ Comets กับ JA
           ├── 🔵 เจ้าหน้าที่ IT (คุณปุ๊ก)
           └── 🔵 เจ้าหน้าที่ IT (คุณยาม้าล)

⚪ พนักงานทั่วไป — ทุกคนในบริษัท
```

### 2.2 ใครทำอะไรได้บ้าง

| ตำแหน่ง | ชื่อ | เห็นอะไร | ทำอะไรได้ |
|---------|-----|----------|-----------|
| 🟣 **ผู้จัดการใหญ่** | คุณเบิร์ด | ทุกเรื่อง ทุกบริษัท | ดูภาพรวม, Dashboard สรุปทั้งหมด |
| 🟠 **หัวหน้า IT** | คุณวี (ICT) / คุณแชมป์ (Comets) | เฉพาะเรื่องในฝั่งตัวเอง | รับเรื่อง, มอบหมายงาน, ลงมือแก้เอง, อนุมัติพนักงานใหม่ |
| 🔵 **เจ้าหน้าที่ IT** | คุณปุ๊ก / คุณยาม้าล | เฉพาะงานที่ถูกมอบหมายให้ | ลงมือแก้, คุยกับพนักงานที่แจ้ง, บันทึกวิธีแก้ |
| ⚪ **พนักงานทั่วไป** | ทุกคน | เฉพาะเรื่องที่ตัวเองแจ้ง | แจ้งปัญหา, เลือกระดับความเร่งด่วน, ยืนยันเมื่อ IT แก้เสร็จ |

### 2.3 การจัดกลุ่ม

ระบบดูที่ **สังกัดของพนักงาน** (บริษัทที่ทำงานอยู่) เพื่อรู้ว่าใครดูแล:
- **ICT** → คุณวี ดูแล
- **Comets + JA** → คุณแชมป์ ดูแล → มอบหมายให้ปุ๊กหรือยาม้าลทำ

## 3. ระบบทำอะไรได้บ้าง?

### 3.1 สิ่งที่มีในเวอร์ชันนี้ (v2.0)

- เข้าใช้งานได้ทั้ง **คอมพิวเตอร์ มือถือ และติดตั้งเป็นแอพ** (PWA) บน Android/iOS
- **ลงทะเบียน/เข้าสู่ระบบ** — สมัครแล้วใช้ได้ทันที ไม่ต้องรออนุมัติ
- **หน้าแรก** แสดงการ์ดใหญ่ 2 ใบ ("แจ้งปัญหา" กับ "ติดตาม Ticket")
- **AI ช่วยแจ้งปัญหา** — พนักงานคุยกับบอท บอทถามข้อมูลครบแล้วสร้าง Ticket ให้
- **ติดตามสถานะ Ticket** ตั้งแต่เปิดจนปิด
- **แชทกับทีม IT** ได้แบบ realtime (ไม่ต้องรีเฟรช)
- **ได้รับแจ้งเตือน** เมื่อมีเรื่องใหม่/สถานะเปลี่ยน (ด้วยกระดิ่ง + ป๊อปอัพ + เสียง)
- **Dashboard สถิติ** สำหรับทีม IT ดูภาพรวมงาน
- **AI เรียนรู้จากประสบการณ์** — เมื่อ IT แก้ปัญหาเสร็จ AI จะจำวิธีแก้ไว้ตอบครั้งต่อไป
- **แยกสิทธิ์การเห็นข้อมูล** ตามตำแหน่ง

### 3.2 สิ่งที่ยังไม่มี (อนาคตอาจเพิ่ม)

- คิดค่าใช้จ่าย/เวลาต่อ Ticket (Cost/Billing)
- เชื่อมต่อกับระบบ HR อัตโนมัติ (ตอนนี้ยังต้อง import ข้อมูลเอง)
- รายงานเชิงลึก (BI Analytics)
- แอพ native บนมือถือ (ตอนนี้ใช้ PWA แทน — ก็เหมือนแอพเกือบทุกอย่าง)

## 4. ฟีเจอร์ละเอียด

### 4.1 ลงทะเบียน / เข้าสู่ระบบ

**เข้าสู่ระบบ**
- ใส่รหัสพนักงาน + รหัสผ่าน
- รหัสผ่านเริ่มต้น = รหัสพนักงาน (แนะนำให้เปลี่ยนหลังเข้าครั้งแรก)
- เข้าสำเร็จ → ไปหน้าแรก (เห็นการ์ด 2 ใบ)

**ลงทะเบียน** (พนักงานใหม่)
- กรอกข้อมูล: รหัสพนักงาน, รหัสผ่าน, บริษัท (Comets/ICT/JA), ชื่อ-นามสกุล, ชื่อเล่น, ตำแหน่ง, อีเมล, เบอร์โทร
- **สมัครแล้วใช้ได้ทันที** ไม่ต้องรอใครอนุมัติ
- หัวหน้า IT จะได้รับแจ้งเตือนว่ามีพนักงานใหม่เข้ามา (แค่บอกให้รู้เฉย ๆ)

**ลืมรหัสผ่าน** → ติดต่อทีม IT ให้ช่วย reset ให้

---

### 4.2 หน้าแรก — 2 การ์ดใหญ่

เมื่อเข้าสู่ระบบแล้ว จะเห็นการ์ดใหญ่ 2 ใบให้เลือก:

| การ์ด | ทำอะไร |
|-------|--------|
| 🎫 **"แจ้งปัญหา"** | เปิด AI Chatbot แบบเต็มจอ — คุยกับบอทเพื่อแจ้งเรื่อง |
| 📋 **"ติดตาม Ticket"** | ดูรายการ Ticket ที่เปิดไว้ สถานะเป็นยังไง |

มีแถบด้านล่างเขียนบอกว่า *"ถ้าอยากกรอกฟอร์มเอง กด 'เปิด Ticket' จากเมนูบนได้"* — เป็นทางเลือกสำรอง

---

### 4.3 AI Chatbot ช่วยแจ้งปัญหา

**คืออะไร**
AI ผู้ช่วยที่ทำหน้าที่**คุยกับพนักงาน เพื่อเก็บข้อมูลปัญหา แล้วสร้าง Ticket ให้อัตโนมัติ** — พนักงานไม่ต้องกรอกฟอร์ม ไม่ต้องรู้ว่าต้องใส่อะไรบ้าง

**บอทจะถามอะไรบ้าง** (5 ข้อ)
1. **ปัญหาที่เจอ** — เช่น เครื่องค้าง, เมลส่งไม่ได้, ปริ้นไม่ออก
2. **อยู่ที่ไหน** — Comets HQ / Comets FAC / ICT / JA / บ้านแสง
3. **จุดที่พบปัญหา** — เช่น "ชั้น 2 ห้อง IT" / "โต๊ะทำงาน"
4. **เร่งด่วนแค่ไหน** — บอทประเมินให้ก่อน แล้วถามยืนยัน
5. **VNC ของเครื่อง** — ถ้ามีให้บอก, ไม่มีข้ามได้

**ขั้นตอน**
1. พนักงานกดการ์ด "แจ้งปัญหา" → เปิดหน้าแชทเต็มจอ
2. บอททักทาย บอกว่า "จะถามข้อมูลให้ครบ"
3. พนักงานเล่าปัญหา → บอทค่อย ๆ ถามเพิ่มตามข้อ 1–5
4. บอทสรุปทั้งหมดให้พนักงานดู → ถามว่า "ยืนยันเปิด Ticket ไหม?"
5. พนักงานยืนยัน → ระบบเปิด**หน้าต่างพรีวิว**ให้ตรวจอีกรอบ (แก้ไข/เพิ่มรูปได้)
6. กด **"ยืนยันเปิด Ticket"** → ระบบสร้าง Ticket, ส่งให้ทีม IT

**บอทห้ามทำอะไร**
- ห้ามแนะนำให้พนักงานทำขั้นตอนที่เสี่ยง เช่น แก้ Registry, Format, Reset network — ให้ IT มาทำเท่านั้น
- ห้ามถาม password หรือ OTP
- ห้ามบอกเวลาแน่นอน ("1-2 วัน") ให้ใช้ "โดยเร็วที่สุด"
- ตอบไทยเท่านั้น

**บอททำอะไรได้**
- แนะนำขั้นตอนปลอดภัย เช่น restart, ตรวจสาย LAN, screenshot error มาส่ง
- ตอบคำถามทั่วไปได้โดยไม่ต้องเปิด Ticket เช่น "ดาวน์โหลด Chrome ที่ไหน"

### 4.4 เปิด Ticket ผ่านฟอร์ม (ทางเลือก)

สำหรับคนที่ไม่อยากคุยกับบอท หรืออยากกรอกเอง — กดเมนู "เปิด Ticket" จากบน

**ต้องกรอก**
- บริษัท/สถานที่ (ระบบเติมให้อัตโนมัติตามสังกัด)
- ประเภทงาน → ประเภทปัญหา → อาการ (เลือกจาก dropdown ต่อเนื่อง)
- โลเคชั่น + โซน/จุด
- **ระดับความเร่งด่วน** — พนักงานเลือกเองจาก 4 ระดับ
- รายละเอียดคำขอ
- เบอร์โทร (เติมให้อัตโนมัติ)
- IP/VNC (ไม่บังคับ)
- แนบรูปได้สูงสุด 5 รูป × 5 MB
- แนบไฟล์เอกสารได้ 5 ไฟล์ × 10 MB (PDF, Word, Excel, ZIP)

**หมายเหตุ:** ฟอร์มจะเซฟข้อมูลให้อัตโนมัติทุก 0.5 วินาที — ถ้าเผลอรีเฟรช ข้อมูลจะยังอยู่ 24 ชั่วโมง

### 4.5 ดู Ticket ของตัวเอง / ทีม

**พนักงานทั่วไปเห็น**
- เฉพาะ Ticket ของตัวเอง
- กรองตามสถานะได้: ทั้งหมด / เปิด / ดำเนินการ / เสร็จ / ยกเลิก
- ค้นหาได้ด้วยเลข Ticket หรือคำในรายละเอียด
- แสดง 100 เรื่องล่าสุด + ปุ่ม "โหลดเพิ่ม"
- เรียงด่วนก่อน → ล่าสุดขึ้นก่อน
- **แสดงเวลาที่เหลือก่อนครบกำหนด** (SLA) — สีแดงถ้าเกิน, เหลืองถ้าใกล้หมด

**ทีม IT เห็น**
- **ผู้จัดการใหญ่ (เบิร์ด)** — Ticket ทุกคน ทุกบริษัท
- **หัวหน้า IT (วี / แชมป์)** — เฉพาะเรื่องของบริษัทที่ดูแล
- **เจ้าหน้าที่ IT (ปุ๊ก / ยาม้าล)** — เฉพาะเรื่องที่ถูกมอบหมายให้ตัวเอง
- ด้านบนมี**ตัวเลขสรุป** (เปิด / ดำเนินการ / เสร็จ / ยกเลิก)
- มีปุ่มแก้ไขแต่ละเรื่อง ✏
- อัปเดตเองทุก 30 วินาที

### 4.6 สถานะของ Ticket

Ticket ทุกเรื่องจะอยู่ในสถานะใดสถานะหนึ่ง:

```
เปิด Ticket  →  กำลังดำเนินการ  →  ดำเนินการเรียบร้อย  →  ปิดงานแล้ว
     │                                     │
     └─────────── ยกเลิก                   └── (ถ้า User กดเปิดใหม่ ↻)
```

**พนักงานทำอะไรได้**
- **ยกเลิก Ticket** — ได้เฉพาะตอนสถานะ "เปิด Ticket" (ยังไม่มี IT รับ)
- **ยืนยันปิดงาน** ✓ — เมื่อ IT แจ้งว่าเสร็จแล้ว พนักงานลองใช้งาน ถ้าหายให้กดยืนยัน
- **เปิดใหม่** ↻ — ถ้ายังไม่หาย กดเปิดใหม่ + ระบุเหตุผล (เวลาจะเริ่มนับใหม่)
- **ถ้าเงียบ 7 วัน** — ระบบถือว่าหายแล้ว ปิดเองอัตโนมัติ

**ทีม IT ทำอะไรได้**
- เปลี่ยนระดับความเร่งด่วน
- เปลี่ยนสถานะ (ยกเว้น "ปิดงานแล้ว" — ต้องให้พนักงานยืนยันเท่านั้น)
- เปลี่ยนผู้รับผิดชอบ (มอบหมายให้คนอื่น)
- **กรอก "วิธีแก้ไขปัญหา"** ตอนปิดงาน → AI จะจำไว้ใช้ต่อ

ทุก Ticket จะมี**ห้องแชท**ระหว่างพนักงานที่แจ้งกับ IT ที่ดูแล

- ปุ่มแชทอยู่มุมซ้าย มี 🔴 แสดงจำนวนข้อความใหม่ที่ยังไม่ได้อ่าน
- เลือก Ticket ที่ต้องการ → เห็นประวัติการคุย
- พิมพ์ตอบได้ทันที — **ข้อความเด้งขึ้นทันทีทั้ง 2 ฝั่ง** (ไม่ต้องรีเฟรช)
- Enter = ส่ง / Shift+Enter = ขึ้นบรรทัดใหม่
- แนบรูปหรือไฟล์ได้สูงสุด 10 MB
- คลิกรูปโปรไฟล์คนที่คุยด้วย → ดูเบอร์โทร + อีเมล ติดต่อต่อได้

### 4.11 Dashboard สรุปสถิติ

เห็นได้เฉพาะ**ทีม IT** (พนักงานทั่วไปไม่เห็น)

**แต่ละคนเห็นไม่เหมือนกัน**
- **ผู้จัดการใหญ่** → ทุกบริษัท
- **หัวหน้า IT** → เฉพาะบริษัทที่ดูแล
- **เจ้าหน้าที่ IT** → เฉพาะงานของตัวเอง

**ข้อมูลที่แสดง**
- การ์ดสรุป: เปิด / ดำเนินการ / รออนุมัติ / เสร็จ
- กราฟ 7+ ตัว:
  - เทรนด์รายเดือน
  - จำแนกตามสถานที่
  - Top 10 ประเภทปัญหาที่เจอบ่อย
  - Top 10 อาการที่เจอบ่อย
  - สรุปผู้รับผิดชอบ
  - แผนกที่แจ้งปัญหาบ่อย
  - ระดับความเร่งด่วน
  - **ตารางวัด performance ของ IT แต่ละคน** (งานที่รับ, งานเสร็จ, ตรงเวลา/เกิน)
  - อัตรางานเสร็จตรงเวลา (SLA)
- **คลิกกราฟ → เปิด popup ดูรายละเอียด Ticket ในกลุ่มนั้น**
- กรองข้อมูลตาม ปี/เดือน/ผู้รับผิดชอบ ได้
- **Export เป็น PDF** ได้

### 4.12 Knowledge Base — คลังความรู้ AI

**ที่มา:** เมื่อทีม IT ปิดงาน + กรอก "วิธีแก้ไขปัญหา" → ระบบจะเก็บไว้เป็นคลังความรู้

**ใช้ทำอะไร:** AI Chatbot จะดึงความรู้เหล่านี้มาตอบคำถามในอนาคต — ยิ่งใช้งานนาน ยิ่งฉลาดขึ้น

**ตัวอย่าง:** ถ้าปุ๊กเคยแก้ "เมล outlook เปิดไม่ได้" ไว้ — ครั้งหน้าถ้าพนักงานถามเรื่องเดียวกัน AI จะรู้ว่าต้องทำอะไร

## 8. เรื่องที่ยังค้างอยู่

1. **คุณแชมป์ (หัวหน้า IT Comets) กับ คุณยาม้าล (เจ้าหน้าที่ IT) ยังไม่มีในระบบ** — ต้องรอสมัครก่อน
2. **ฝั่ง ICT ไม่มีเจ้าหน้าที่** — คุณวีต้องลงมือเองทุกเรื่อง อยากเพิ่มคนช่วยไหม?
3. **Ticket เกินเวลา** ตอนนี้แค่แจ้งเตือน — อนาคตอยากให้ส่งต่อคนอื่นอัตโนมัติไหม?
4. **หน้าจัดการตำแหน่ง** (Role) — ตอนนี้ถ้าจะเปลี่ยนตำแหน่งใครต้องแก้ในฐานข้อมูลตรง ๆ อยากมีหน้าเว็บไหม?
5. **ลงทะเบียนด้วย OTP** — ตอนนี้ใครกรอกข้อมูลก็สมัครได้เลย อยากเพิ่มการยืนยันไหม?

## 9. คำศัพท์ที่ใช้

| คำ | หมายถึง |
|----|---------|
| **Ticket** | เรื่องแจ้งปัญหา 1 เรื่อง (เลขเริ่มด้วย `IT` เช่น `IT26040288`) |
| **Scope** | ขอบเขตที่แต่ละคนเห็นข้อมูลได้ (ICT / Comets / ทั้งหมด) |
| **SLA** | เวลาที่ต้องเสร็จภายใน ตามระดับความเร่งด่วน |
| **ผู้รับผิดชอบ** (Assignee) | คนที่ถูกมอบหมายให้แก้ Ticket นั้น ๆ |
| **วิธีแก้** (Solution) | ขั้นตอนที่ IT บันทึกไว้ตอนปิดงาน |
| **KB** (Knowledge Base) | คลังความรู้ที่ AI ใช้เรียนรู้ |
| **PWA** | เว็บที่ติดตั้งเป็นแอพบนมือถือได้ |
| **Auto-close** | ระบบปิดเองอัตโนมัติเมื่อพนักงานเงียบ 7 วัน |
| **Auto-approve** | อนุมัติอัตโนมัติตอนลงทะเบียน ไม่ต้องรอใคร |
