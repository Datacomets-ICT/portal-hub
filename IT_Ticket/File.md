## 1. Overview

IT Ticket System เป็นระบบแจ้งปัญหาและติดตามสถานะงาน IT สำหรับองค์กร ใช้เป็นช่องทางหลักในการรับเรื่อง แจ้งปัญหา แชทกับทีม IT และติดตาม SLA แบบออนไลน์ รองรับทั้งผู้ใช้งานทั่วไป (พนักงาน) และทีม IT (Admin) และสามารถติดตั้งเป็น PWA บนมือถือได้ [file:1]

### 1.1 Objectives

- ลดการใช้ช่องทางแจ้งปัญหาแบบกระจัดกระจาย (โทร, ไลน์, ปากเปล่า) รวมศูนย์ผ่านระบบเดียว. [file:1]
- เพิ่มความโปร่งใสในการติดตามสถานะงาน IT ด้วย SLA countdown และสถานะ Ticket ที่ชัดเจน. [file:1]
- ลดภาระทีม IT ด้วยการใช้ AI Chatbot ช่วยตอบคำถามเบื้องต้นและสร้าง Ticket อัตโนมัติ. [file:1]
- เก็บประวัติการแก้ไขปัญหาเป็น Knowledge Base ให้ AI เรียนรู้และให้คำตอบได้ดีขึ้นในอนาคต. [file:1]

### 1.2 Target Users

- พนักงานทั่วไป (User) ที่ต้องการแจ้งปัญหาและขอความช่วยเหลือด้าน IT. [file:1]
- ทีม IT / แอดมิน (Admin) ที่รับผิดชอบดูแล Ticket, จัดลำดับความสำคัญ และสื่อสารกับผู้ใช้. [file:1]
- ผู้บริหาร/หัวหน้า (ในบางเคส) ที่ต้องอนุมัติ Ticket ประเภทที่ต้องการ Approve. [file:1]

## 2. Scope

### 2.1 In Scope

- ระบบ Web สำหรับ Desktop/Mobile (Responsive) พร้อมความสามารถติดตั้งเป็น PWA. [file:1]
- การจัดการ Ticket ตั้งแต่เปิด Ticket, ดำเนินการ, ปิดงาน, ยกเลิก, และ Auto-close. [file:1]
- ระบบแชท Realtime ระหว่าง User และทีม IT ภายใน Ticket. [file:1]
- ระบบแจ้งเตือน (Notification Panel + Popup + Badge + เสียง beep). [file:1]
- ระบบ AI Chatbot สำหรับตอบคำถาม IT เบื้องต้นและเปิด Ticket อัตโนมัติ. [file:1]
- ระบบอนุมัติผู้ใช้งานใหม่และสิทธิ์ Admin. [file:1]

### 2.2 Out of Scope (เบื้องต้น)

- การคิดต้นทุน/ค่าใช้จ่าย IT ต่อ Ticket (Costing/Billing).
- การเชื่อมต่อกับ HR System นอกเหนือจากการอ่านข้อมูลรหัสพนักงานแบบ manual/import (ถ้ามี).
- Dashboard วิเคราะห์เชิงลึก (BI) นอกเหนือจากสรุปตัวเลขพื้นฐานในหน้ารายการ Ticket. [file:1]

## 5. Core Features – User Side

### 5.1 เปิด Ticket ใหม่

#### Requirement

- ผู้ใช้งานสามารถเปิด Ticket สำหรับแจ้งปัญหาหรือขอความช่วยเหลือด้าน IT. [file:1]

#### Functional Details

- เมนู: "เปิด Ticket" บน Navbar. [file:1]
- ฟอร์มประกอบด้วย:
  - บริษัท/สถานที่ (Comets, ICT, JA) – Default ตามสังกัดผู้ใช้. [file:1]
  - ประเภทงาน (คอมพิวเตอร์, ปริ้นเตอร์, ปัญหาโปรแกรม, ขอสิทธิ์เข้าระบบ ฯลฯ). [file:1]
  - ประเภทปัญหา / อาการ (จาก dropdown ที่ระบบกำหนด). [file:1]
  - โลเคชั่น (Comets HQ, Comets FAC, ICT, JA, บ้านแสง + ระบุโซน/จุด). [file:1]
  - รายละเอียดคำขอ (ข้อความอิสระ). [file:1]
  - Priority: ด่วนมาก / สำคัญ / ปกติ / ไม่เร่ง. [file:1]
  - แนบรูป: สูงสุด 5 รูป × 5 MB. [file:1]
  - แนบไฟล์เอกสาร: สูงสุด 5 ไฟล์ × 10 MB (PDF, Word, Excel, ZIP). [file:1]
- เมื่อกด "ส่ง Ticket":
  - ระบบสร้างเลข Ticket เช่น IT26040288. [file:1]
  - แสดง progress bar ระหว่างอัปโหลดไฟล์. [file:1]

#### Special Behaviors

- Auto-save ฟอร์มทุก 0.5 วินาที เก็บข้อมูลใน Browser (เช่น localStorage) ได้นาน 24 ชม. [file:1]
- หากส่งไม่สำเร็จ:
  - ไฟล์ที่อัปโหลดสำเร็จแล้วไม่ต้องอัปใหม่. [file:1]
  - มีปุ่ม "ลองส่งใหม่" ให้กดซ้ำได้ทันที. [file:1]

### 5.2 Ticket List (ของตัวเอง)

- เมนู "Ticket List" แสดงเฉพาะ Ticket ของผู้ใช้คนนั้น. [file:1]
- Filter ด้านบน: ทั้งหมด / เปิด Ticket / กำลังดำเนินการ / เสร็จเรียบร้อย / ยกเลิก. [file:1]
- ช่องค้นหา: ค้นด้วยเลข Ticket, ประเภท, รายละเอียด. [file:1]
- คอลัมน์สำคัญ:
  - Priority (ไอคอนสี 🔴🟠🟡⚪ และเรียง Urgent มาก่อน). [file:1]
  - SLA countdown (เวลาที่เหลือก่อนครบกำหนด คิดเฉพาะเวลาทำงาน จ–ศ 08:00–17:00; เหลือน้อย = สีเหลือง, เกินกำหนด = สีแดง). [file:1]
  - สถานะ (เปิด Ticket, กำลังดำเนินการ, ดำเนินการเรียบร้อย, ปิดงานแล้ว, ยกเลิก). [file:1]
- Pagination: โหลด Ticket ล่าสุด 100 รายการ และมีปุ่ม "โหลดเพิ่ม". [file:1]
- ปุ่ม "ดู" แสดง Modal รายละเอียด Ticket รวมรูปแนบและข้อความแชท. [file:1]

### 5.3 ยกเลิก Ticket

- เฉพาะ Ticket ที่สถานะ "เปิด Ticket" เท่านั้น. [file:1]
- ผู้ใช้กดปุ่ม ❌ สีแดง ในแถว Ticket ที่ต้องการ. [file:1]
- ระบบถามยืนยัน และเปลี่ยนสถานะเป็น "ยกเลิก" ทันทีหลังยืนยัน. [file:1]

### 5.4 ยืนยันปิดงาน / เปิด Ticket ใหม่

- เมื่อทีม IT เปลี่ยนสถานะเป็น "ดำเนินการเรียบร้อย" ระบบรอให้ User ยืนยัน. [file:1]

กรณี 1: งานเสร็จเรียบร้อย  
- ผู้ใช้ไปที่ Ticket List เลือก Ticket ที่สถานะ "ดำเนินการเรียบร้อย". [file:1]
- กดปุ่ม ✓ สีเขียว "ยืนยันปิดงาน". [file:1]
- สถานะเปลี่ยนเป็น "ปิดงานแล้ว" และแจ้งเตือนทีม IT. [file:1]

กรณี 2: ปัญหายังไม่หาย / ต้องการเปิดใหม่  
- กดปุ่ม สีส้ม "ยังไม่เสร็จ/เปิดใหม่". [file:1]
- ระบุเหตุผล เช่น "ปัญหากลับมาเกิดอีก" หรือ "วิธีแก้ที่ให้มาไม่ได้ผล". [file:1]
- กด "ยืนยันเปิดใหม่" สถานะกลับเป็น "เปิด Ticket" และแจ้งทีม IT. [file:1]

Auto-close:  
- หาก User ไม่กดยืนยันภายใน 7 วัน ระบบปิด Ticket อัตโนมัติเป็น "ปิดงานแล้ว". [file:1]
- ใน Modal รายละเอียด Ticket แสดงแถบ countdown (สีฟ้า, ถ้าเหลือ ≤2 วันเปลี่ยนเป็นสีส้ม). [file:1]

## 6. AI Chatbot & Chat System

### 6.1 AI Chatbot (IT Support)

#### Use Case

- ให้ User ถามปัญหา IT เบื้องต้นได้ตลอด 24 ชั่วโมงและช่วยลดจำนวน Ticket ที่ไม่จำเป็น. [file:1]

#### Behavior

- ปุ่มแชทสีม่วง "IT Support" อยู่ด้านซ้ายของหน้าจอ. [file:1]
- เมื่อเปิดกล่องแชท ผู้ใช้สามารถพิมพ์คำถาม เช่น "คอมค้าง", "ปริ้นไม่ออก", "อยากลง VS Code", "wifi เชื่อมไม่ได้". [file:1]
- AI ตอบขั้นตอนการแก้ไข. [file:1]
- หากปัญหายังไม่หาย ผู้ใช้พิมพ์ "ไม่หาย" หรือข้อความที่สื่อความหมาย ระบบจะ:
  - เปิด Ticket ใหม่อัตโนมัติ.
  - ดึงข้อมูลจากบทสนทนาไปเติมฟิลด์ในฟอร์ม Ticket. [file:1]
- Knowledge Base:
  - เมื่อ Admin ปิด Ticket โดยใส่ "วิธีแก้ไขปัญหา" ระบบจะเพิ่มข้อมูลนี้เข้า Knowledge Base ให้ AI เรียนรู้. [file:1]

### 6.2 Chat ระหว่าง User – IT ใน Ticket

- มีปุ่มแชทสีเขียวด้านซ้ายมือ เมื่อมีข้อความใหม่ badge สีแดงแสดงจำนวนข้อความที่ยังไม่ได้อ่าน. [file:1]
- ภายในกล่องแชท แสดงรายการ Ticket ที่มีการสนทนา. [file:1]
- ผู้ใช้เลือก Ticket เพื่อดูประวัติการแชท และสามารถพิมพ์ตอบ แนบรูป/ไฟล์ (สูงสุด 10 MB ต่อไฟล์). [file:1]
- ระบบส่งข้อความแบบ Realtime (เด้งทันทีไม่ต้อง refresh; Enter = ส่ง, Shift+Enter = ขึ้นบรรทัดใหม่). [file:1]
- ข้อความแชทถูกเก็บในตาราง `ticket_messages` ใน Supabase แบบถาวร. [file:1]

### 6.3 Contact Popup

- ในรายการแชท Ticket (ด้านซ้าย) เมื่อคลิกที่ avatar ของอีกฝ่าย จะเปิด popup แสดง:
  - ชื่อ, เบอร์โทร, อีเมล. [file:1]
- คลิกที่เบอร์โทร:
  - บนมือถือ: เปิดหน้าต่างโทรออกทันที.
  - บน Desktop: copy หรือเปิดแอปโทรตามค่า default. [file:1]
- คลิกอีเมล: เปิดโปรแกรมส่งเมลพร้อมกรอกปลายทาง. [file:1]

## 9. Admin Features

### 9.1 Ticket Management

#### ดู Ticket ของทุกคน

- เมนู "Ticket List" สำหรับ Admin แสดง Ticket ทั้งหมดทุกคน. [file:1]
- มี Summary ตัวเลขด้านบน แสดงจำนวน Ticket แยกตามสถานะ. [file:1]
- Auto-refresh ทุก 30 วินาที. [file:1]

#### แก้ไข Ticket

- คลิกไอคอนดินสอสีเขียว ✏ เพื่อเปิด Modal แก้ไข. [file:1]
- ใน Modal:
  - ข้อมูล Ticket: ผู้แจ้ง, สถานที่, ประเภท, รายละเอียด. [file:1]
  - Dropdown "ผู้รับผิดชอบ" (รายชื่อทีม IT ที่ is_admin = true). [file:1]
  - ปรับ Priority (SLA deadline จะคำนวณใหม่อัตโนมัติ). [file:1]
  - เปลี่ยนสถานะ: เปิด Ticket, กำลังดำเนินการ, ต้องการ Approve, ดำเนินการเรียบร้อย, ยกเลิก. [file:1]
  - ช่อง "วิธีแก้ไขปัญหา" สำหรับบันทึก solution (ให้ AI เรียนรู้). [file:1]
  - ประวัติการแก้ไข (ใคร แก้อะไร เมื่อไหร่). [file:1]
- กด "บันทึก" เพื่อบันทึกการเปลี่ยนแปลงและแจ้ง User. [file:1]

Notes:  
- สถานะ "ปิดงานแล้ว" เปลี่ยนได้โดย User เท่านั้น หรือระบบ Auto-close หลัง 7 วัน. [file:1]

### 9.2 Assignment

## 11. Non-functional Requirements (Draft)

(ส่วนนี้เป็นข้อเสนอเพิ่มเติม สามารถปรับได้ตาม requirement ของคุณ)

- Performance:
  - โหลด Ticket List 100 รายการล่าสุดได้ภายใน ≤ 2 วินาที บนเครือข่ายภายในองค์กร. [file:1]
- Reliability:
  - ระบบต้องสามารถทำงานได้อย่างต่อเนื่องในเวลาทำการ จ–ศ 08:00–17:00 อย่างน้อย 99%. [file:1]
- Security:
  - แยกสิทธิ์การเข้าถึง User/Admin ตามที่ระบุใน Permission Matrix.
  - ป้องกันการเข้าถึง Ticket ของผู้อื่นโดยไม่ได้รับอนุญาต.
- Data:
  - เก็บประวัติ Ticket และข้อความแชทใน Supabase อย่างถาวร (ไม่มีการลบอัตโนมัติ ณ เวอร์ชันนี้). [file:1]
- UX:
  - รองรับภาษาไทยเป็นหลัก.
  - ใช้สี Priority/Status ให้เข้าใจง่ายและสอดคล้องกันทั้งระบบ. [file:1]
