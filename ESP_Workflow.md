# ESP (Employee Service Portal) — Workflow

> เอกสารนี้อธิบาย flow การทำงานของระบบ ESP ตั้งแต่ login → ใช้งานแต่ละ service
> **Phase 1:** จองห้องประชุม (Meeting Room Booking) — กำลังพัฒนา
> **Phase 2–4:** IT Ticket (มีอยู่แล้ว), จองรถ, แจ้งซ่อม

---

## 📌 ภาพรวมระบบ

```
┌─────────────────────────────────────────────────────┐
│              ESP — Employee Service Portal          │
│                 (login ครั้งเดียว)                    │
└─────────────────────────────────────────────────────┘
                         │
         ┌───────────────┼───────────────┬─────────────┐
         ▼               ▼               ▼             ▼
   ┌──────────┐   ┌──────────┐    ┌──────────┐   ┌──────────┐
   │IT Ticket │   │ห้องประชุม│    │ จองรถ    │   │ แจ้งซ่อม │
   │(Phase 1✓)│   │(Phase 1)│    │(Phase 3) │   │(Phase 4) │
   └──────────┘   └──────────┘    └──────────┘   └──────────┘
```

**Roles:**
- **User** = พนักงานทั่วไป (ใช้บริการ)
- **Admin** = ผู้ดูแลระบบแต่ละ service (IT admin, ผู้ดูแลห้อง, ผู้ดูแลรถ, ช่างซ่อม)
- **Super Admin** = ดูแลทั้งระบบ + จัดการ master data

### 3.3 User Actions ทั้งหมด
| Action | รายละเอียด |
|--------|-------------|
| ดูห้องทั้งหมด | เห็นสถานะ available/occupied real-time |
| ค้นหา/กรอง | ตาม ความจุ, อุปกรณ์, สถานที่ (Comets HQ/ICT) |
| ดู availability | ปฏิทินรายห้อง (รายวัน/รายสัปดาห์) |
| จองห้อง | ฟอร์มเต็ม หรือ Quick Book |
| ยกเลิกการจองตัวเอง | ได้เฉพาะก่อนเวลาเริ่ม |
| แก้ไขการจอง | เปลี่ยนเวลา/ผู้เข้าร่วม/รายละเอียด |
| Check-in / Check-out | (อนาคต) QR code หน้าห้อง |
| ดูประวัติ | การจองทั้งหมดของฉัน |

```

   │
   ▼

│  Dashboard — จัดการห้องประชุม         │

│  ├─ การจองวันนี้: 15                  │
│  ├─ ห้องที่ใช้บ่อย: NORTH STAR       │
│  ├─ Peak hour: 10:00–11:00            │
│  └─ Occupancy rate: 68%               │
│                                         │
│  🔔 รอดำเนินการ                        │
│  ├─ คำขอพิเศษ: 2                      │
│  └─ ห้อง maintenance: 1               │
│                                         │
│  🏢 จัดการห้อง                         │

│  (timeline ห้องทั้งหมดในวันเดียว)     │

```

### 4.2 Admin Actions ทั้งหมด
| หมวด | Action |
|------|---------|
| **จัดการห้อง** | เพิ่ม/แก้ไข/ลบ ห้อง, อัปโหลดรูป, แก้ความจุ/อุปกรณ์ |
| **สถานะห้อง** | เปิด/ปิดปรับปรุง (maintenance mode) |
| **จัดการการจอง** | ดูการจองทั้งหมด, override, ยกเลิกให้ user, แก้ไข |
| **รายงาน** | Export Excel/PDF, สถิติการใช้งาน, peak hours |
| **อนุมัติ** | คำขอห้องพิเศษ (ถ้าเปิดโหมดต้องอนุมัติ) |
| **ประกาศ** | โพสต์ประกาศห้องปิด/อีเวนต์พิเศษ |
| **Audit log** | ดูประวัติการแก้ไขทุกอย่าง |

### 4.3 Flow อนุมัติ (ถ้าเปิด approval mode)
```
User จองห้อง
   │
   ▼ (ถ้าห้องต้องอนุมัติ เช่นห้อง TownHall, ห้องต้อนรับ)
สถานะ: "รออนุมัติ"
   │
   ▼
Admin เห็นใน queue
   │
   ├──[อนุมัติ]───▶ สถานะ: confirmed + แจ้ง user
   └──[ปฏิเสธ]───▶ สถานะ: rejected + เหตุผล + แจ้ง user
```

## 🔔 6. Notification Flow

**User ได้รับแจ้งเตือนเมื่อ:**
- จองสำเร็จ
- การจองถูก admin ยกเลิก/แก้ไข
- ก่อนเวลาประชุม 15 นาที (reminder)
- มีคนเชิญเข้าร่วมประชุม (ถ้าเปิด invite)

**Admin ได้รับแจ้งเตือนเมื่อ:**
- มีคำขอจองที่ต้องอนุมัติ
- มี conflict / overbooking
- ห้องที่ดูแลมีปัญหา (user รายงาน)

## ❓ คำถามต้องการ confirm

1. **ชื่อระบบ** — ESP / Comets Portal / อื่น ๆ?
2. **Admin ห้องประชุม** — ใครเป็นบ้าง? (ใช้ field เดิม `role='admin'` หรือแยก role ใหม่?)
3. **Approval mode** — ทุกห้องต้องอนุมัติ / เฉพาะห้องใหญ่ / ไม่ต้อง?
4. **Business hours** — จองได้ช่วงไหน? (เวลาทำงาน 8:00–17:00 เท่านั้น หรือ 24 ชม.?)
5. **Max duration** — จองได้นานสุดกี่ชั่วโมง/วัน?
6. **Recurring booking** — จองประชุมซ้ำทุกสัปดาห์ได้มั้ย?
