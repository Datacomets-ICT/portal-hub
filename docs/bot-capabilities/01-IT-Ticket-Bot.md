# IT-Ticket Chatbot

> AI ผู้ช่วยพนักงานเปิด IT Ticket — แชทคุยกันธรรมดา ระบบเก็บข้อมูลเองให้ครบ แล้วเปิด ticket draft ให้กดยืนยัน

---

## บอททำอะไรได้บ้าง

1. **คุยกันแบบธรรมชาติ** — user พิมพ์ "เมาส์ไม่ติด" / "Email เข้าไม่ได้" ก็พอ ไม่ต้องกรอกฟอร์ม
2. **ถามทีละข้อจนครบ 5 อย่าง** — อาการ / สถานที่ / ชั้น / แผนก / ระดับเร่งด่วน
3. **อ่านรูปที่แนบ** — เห็น BSOD ก็รู้เลยว่า "หน้าจอฟ้า" ไม่ต้องถามเพิ่ม
4. **ค้น knowledge base** — ดึงคำตอบจากคู่มือ/ticket เก่ามาประกอบ
5. **เปิดฟอร์ม draft อัตโนมัติ** — ส่งให้ user ตรวจก่อน submit เข้าระบบจริง

---

## ใช้ AI กี่ตัว ทำไมเยอะ

ใช้ทั้งหมด **4 ตัวเรียงลำดับ** — ตัวบนพังก็ตกลงตัวล่างทันที (เหมือนสำรองไฟ ถ้าไฟดับก็ใช้ generator)

### หลัก: Groq Scout

ใครเป็นเจ้าของ: บริษัท Groq (USA) — เน้น chip ทำ AI inference เร็วๆ
ใช้รุ่น: Llama 4 Scout (Meta สร้าง 17 พันล้าน parameters)

**เก่งเรื่อง**
- ภาษาไทยดีมาก เข้าใจสำนวน, คำแสลง, ประโยคยาว
- ทำตามคำสั่งเป๊ะ — ลิสต์ตัวเลือก, ห้ามเดา, format ตามแบบ
- เร็ว — ตอบใน 1-2 วินาที
- อ่านรูปได้

**จุดอ่อน:** quota ฟรีเล็ก — 30,000 token/นาที = 1 บทสนทนา/นาที พอเต็มก็ตกชั้น

### สำรอง 1: Cerebras

ใครเป็นเจ้าของ: บริษัท Cerebras (USA) — ผลิต chip AI ที่ใหญ่ที่สุดในโลก
ใช้รุ่น: Llama 3.1 (8 พันล้าน parameters — เล็กกว่า)

**เก่งเรื่อง**
- เร็วที่สุดในตลาด ~2,000 ตัวอักษร/วินาที (Groq ก็ ~1,500)
- quota ฟรีใหญ่กว่า Groq 2 เท่า (60,000 token/นาที)

**จุดอ่อน:** เป็น model เล็กกว่า — บางครั้งภาษาไทยเพี้ยน, เข้าใจ context ซับซ้อนได้น้อยกว่า

### สำรอง 2: Gemini 2.0 Flash

ใครเป็นเจ้าของ: Google
ใช้รุ่น: Gemini 2.0 Flash (multi-modal — รับรูป/วิดีโอ/audio ได้)

**เก่งเรื่อง**

- ภาษาไทยดี

**จุดอ่อน:** ช้ากว่า Groq/Cerebras (~500 ตัวอักษร/วินาที), format อาจไม่ตรงเป๊ะ

ใครเป็นเจ้าของ: Groq (เหมือนตัวหลัก)
ใช้รุ่น: Llama 3.1 8B Instant

## OCR — อ่านข้อความจากรูป

เมื่อ user แนบรูป (เช่น screenshot error, รูปหน้าจอฟ้า) ระบบจะอ่านข้อความออกมาให้ AI เห็น

**ลำดับ:**
1. **Gemini Vision** อ่านก่อน — เก่งมาก
2. ถ้าอ่านไม่ออก → **Groq Vision (llama-4-scout)** ลองต่อ — เผื่อ Gemini พลาด

แม้ทั้ง 2 อ่านไม่ออก bot ก็ฉลาดพอจะถาม "ใช้เครื่องอะไร?" (PC/Notebook/Macbook/iMac) แทนการถามแบบเปิด

---

## ค่าใช้จ่ายต่อบทสนทนา (Token)

แต่ละครั้งที่ user พิมพ์ ระบบส่งข้อมูลประมาณ **2,500 token** ไปให้ AI (รวมคำถาม + system prompt + ประวัติ + worklist)

**บทสนทนา 1 ใบ (7 turn) ใช้ ~19,000 token**

ถ้ามี user 100 คน × วันละ 1 ticket = **1.9 ล้าน token/วัน** หรือ **~57 ล้าน token/เดือน**

---

## ค่าใช้จ่ายจริง (ตอนนี้กับถ้า upgrade)

### ตอนนี้ใช้ Free tier ทั้งหมด — **0 บาท/เดือน**

quota ฟรีของ 4 providers รวมกัน = ~1.1 ล้าน token/นาที — รับ user หลายร้อยคนได้สบาย

### ถ้าอยากไป Paid tier (เผื่ออนาคต volume สูงขึ้น)

| Provider | Paid plan | ราคา | ได้อะไรเพิ่ม |
|---|---|---|---|
| **Groq Pro** | Pay-as-you-go | $0.18/$0.59 ต่อล้าน token (input/output) | quota เพิ่ม 10x, ความเสถียร SLA |
| **Cerebras Pro** | Pay-as-you-go | ~$0.10 ต่อล้าน token | quota เพิ่ม, priority routing |
| **Gemini Pro Tier 1** | Pay-as-you-go | $0.075/$0.30 ต่อล้าน token | 2,000 req/นาที (เพิ่มจาก 15) |

### ตัวอย่างคิดเงินสำหรับ Comets (สมมติ 57M token/เดือน)

ถ้าใช้ Groq Pro ทั้งหมด = 57M × ($0.18 + $0.59)/2 ≈ **$22/เดือน (~770 บาท)**
ถ้าใช้ Gemini Pro ทั้งหมด = 57M × ($0.075 + $0.30)/2 ≈ **$11/เดือน (~385 บาท)**

**สรุป:** ถ้า user ขยายเป็น 1,000 คน ค่า AI ก็ยังไม่เกิน 1,000 บาท/เดือน
